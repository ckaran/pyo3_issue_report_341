# Example project for bug report

This is not a real project; it exists only to make it easier to explain the
issue for https://github.com/PyO3/pyo3/issues/341.  Do not attempt to use this
project for useful work.

## Bug report details

To trigger this bug, go into the `Cargo.toml` file and change the following
line:

```
pyo3 = {version="=0.6.0-alpha.1", features = ["extension-module"] }
```

to

```
pyo3 = {version="=0.6.0-alpha.2", features = ["extension-module"] }
```

You will also need to be using rust nightly.

Once you do that, `cargo -vvv test` should give output similar to that at the
end of this file.  `python3 setup.py build` seems to work under alpha 2, but I
can't load the module under alpha 2; within the python3 REPL I get the following
error:

```
cfkaran2@EMANE:~/Desktop/pyo3_issue_report_341/target/release$ python3
Python 3.7.1 (default, Nov 13 2018, 16:55:45)
[GCC 7.3.0] on linux
Type "help", "copyright", "credits" or "license" for more information.

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ModuleNotFoundError: No module named 'py_disc_model.so'; 'py_disc_model' is not a package
```

Under `0.6.0-alpha.1`, I can use `cargo -vvv test` without any problems, and
when I use `python3 setup.py build`, I can load the module into python
correctly.
